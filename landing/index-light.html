01 / The problem

Coding agents are brilliant at typing.
Less so at shipping.

Today's coding agents handle the easy part — writing code. The rest of the software lifecycle is glued together with brittle scripts, unscoped tokens, and ad-hoc prompts. AgentRail is the open-source layer that fills the gap.

i.
PROBLEM

No lifecycle, just a prompt and a prayer.

Agents lose context between turns, retry on the wrong commits, and re-ask the same question because nothing tracks state.

ii.
PROBLEM

Personal access tokens with full repo scope.

Most agent stacks reuse the developer's GitHub PAT — meaning every run holds keys to every repository in the org.

iii.
PROBLEM

CI is a wall, not a feedback loop.

Test failures arrive as opaque logs ten minutes after the agent moved on, with no structured signal to act on.

02 / The platform

Eight primitives.
One open-source backbone.

AgentRail is the open-source middle layer between your agent and the rest of software delivery. Run it locally, extend it freely, and own your data. Each primitive is a narrow API with a typed SDK on top.

Tasks

Structured lifecycle objects with state, retries, and idempotent transitions. The unit of work for every agent.

POST /v1/tasks

Runs

Sandboxed execution contexts with provisioned environments, timeouts, and replayable transcripts.

POST /v1/runs

CI Feedback

Structured test, lint, and build results streamed to the agent as typed events — not log scraping.

GET /v1/runs/:id/checks

Scoped Auth

Credential policies are modeled per task — repo, branch, and label scopes are explicit so org-wide tokens can be phased out as adapters mature.

POST /v1/credentials

Event Stream

Server-sent events for every state change. Pipe agent activity into Slack, your IDE, or your own observability stack.

GET /v1/events

Webhooks

Typed delivery with retries, signatures, and replay. Wire AgentRail into the rest of your delivery pipeline.

POST /v1/webhooks

Audit Log

Every action taken, every token used, every file touched — append-only and exportable.

GET /v1/audit

Replay & Rollback

Re-run any task from any prior state with the exact same inputs. Roll back a merge with one call.

POST /v1/tasks/:id/replay
03 / Task lifecycle

Five stations.
One typed rail.

Every task on AgentRail moves through five well-defined stages. Transitions are atomic, observable, and reversible — agents can fail loudly, retry safely, and never lose context mid-flight.

01

Queue — everything starts as an intent

Tasks enter the queue from anywhere — a webhook, a Slack thread, a Linear ticket, an SDK call. Each gets a stable ID, a typed payload, and a deterministic priority. No work happens until the queue dispatches. 

// queue a task from anywhere
const task = await agentrail.tasks.create({
  agent: "claude-code",
  repo:  "acme/payments",
  intent: "migrate auth handler to v2 API",
  priority: "normal",
});
// → task.id = "tsk_8Hk2..."  state = "queued"
04 / SDK

A typed SDK,
not a wrapper.

First-class TypeScript and Python clients with full schema generation, streaming, retries, and idempotency baked in. Every event is a typed discriminated union — no any escape hatches. No API key required for local runs.
